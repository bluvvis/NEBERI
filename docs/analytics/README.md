# Аналитика (ТЗ, US, UC, ПМИ)

Критерии курса: до **15 баллов** за аналитику (`docs/lectures_extracted/vse_o_kurse.md`).

## Рекомендуемый набор артефактов

1. **Краткое ТЗ** (`docs/analytics/TZ.md`) — цель модуля, границы MVP, NFR (latency, доступность), интеграция (REST), ПДн.
2. **User Stories** — шаблон ниже, по 1–2 на экран/API.
3. **Use Cases** — поток: «система BSS шлёт событие → NeBeri → ответ с score + reasons».
4. **ПМИ** — таблица сценариев: входные данные → ожидаемый `risk_level` / код ответа.
5. **Offline-оценка / ML без долга в API** — [`OFFLINE_EVALUATION.md`](OFFLINE_EVALUATION.md) (датасеты, шаблон пайплайна, формулировка для защиты).

## Шаблон User Story

```text
Как <роль: интегратор BSS / аналитик SOC>
Я хочу <действие: отправить событие и получить скор с объяснением>
Чтобы <ценность: автоматизировать первичную фильтрацию мошенничества>
Критерии приёмки:
- OpenAPI описывает POST /v1/events
- В ответе есть policy_version и reasons[] (при наличии в политике — references[] с https-ссылками)
- Номера в ответе замаскированы
```

Заполните реальные US в отдельных файлах `US-NNN-<slug>.md` по мере спринтов.
