# Сигналы скоринга и открытые источники

Таблица связывает **правила в `apps/api/policies/default_rules.yaml`** с тем, что обычно учитывают в **исследованиях и практике** антиспама / smishing в телекоме (без претензии на внутренние регламенты операторов).

Те же ссылки продублированы в YAML как **`references`** и уходят в API в **`reasons[].references`** (title, `https` URL, optional `kind`) — чтобы объяснимость была не только в Markdown, но и в машиночитаемом ответе.

| `rule_id` (YAML) | Смысл сигнала | Открытые ориентиры (что читали / на что опирались) |
|------------------|---------------|-----------------------------------------------------|
| `coercion_urgency` | Давление «срочно / немедленно» в SMS/ASR-тексте | Типичные паттерны социальной инженерии в smishing; обзоры двухступенчатых SMS-систем (напр. [TFC / Springer](https://link.springer.com/chapter/10.1007/978-981-16-9229-1_10)) |
| `scam_keywords` | Лексика «перевод / блокировка / код из смс / банк» | Публичные датасеты и разметка spam vs ham ([UCI SMS Spam](https://archive.ics.uci.edu/dataset/228/sms+spam+collection)); smishing-разметка ([Mendeley dataset](https://data.mendeley.com/datasets/f45bkkt8pr/1)) |
| `night_window_call` | Звонок в ночное окно (UTC MVP) | Эвристика «нестандартное время» в CDR-аналитике; обсуждается в работах по SMS/звонковому фроду |
| `ultra_short_call` | Очень короткий звонок | Паттерн прозвона / сканирования в CDR; см. обзоры hybrid ML + правила в телеком-фроде |
| `burst_short_dials` | Серия коротких вызовов на один номер | Поведенческие фичи по CDR (частота, окно времени) — стандартный класс сигналов в антифроде |
| `high_velocity_sms` | Много SMS с одного номера за окно | Аналог «velocity» / anomaly на исходящем трафике |

**Пороги `risk_level`** (сумма весов, cap 100) заданы в `app/services/scoring.py` и согласованы с демо-весами правил — это **не** калиброванная вероятность, а **объяснимый** индекс для MVP.

## Как обновлять политику

1. Править `default_rules.yaml`, поднять поле **`version`**.
2. Зафиксировать изменение в [`apps/api/policies/CHANGELOG.md`](../apps/api/policies/CHANGELOG.md).
3. Прогнать `pytest apps/api/tests` (в т.ч. `test_policy.py`, `test_events_api.py`).
