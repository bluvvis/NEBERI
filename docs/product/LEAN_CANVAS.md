# Lean Canvas — NeBeri Fraud Signal

> Заполните под защиту (критерий до **15 баллов**, `vse_o_kurse.md`). Ниже — черновик структуры по слайдам курса (`Lean_Canvas.md`).

## 1. Проблемы / боль клиента

- Рост мошеннических звонков/SMS и социальная инженерия.
- Классические фильтры дают мало **объяснимости** для абонента и интеграторов.

## 2. Сегменты клиентов

- **B2B:** продуктовые команды оператора (BSS/OSS, антифрод-платформа).
- **Косвенно:** конечный абонент получает сигнал/подсказку через каналы оператора.

## 3. Уникальное ценностное предложение

Формула из лекции: *«Мы помогаем клиентам X в ситуации Z решать проблему P с помощью O и получить ценность V».*

- **X:** команды телеком-продукта / интеграторы.
- **Z:** поток событий связи в реальном времени.
- **P:** быстро оценить риск и **объяснить почему**.
- **O:** NeBeri API + политика правил + консоль/интеграция.
- **V:** снижение потерь, доверие к сигналам, скорость реакции.

## 4. Решение (MVP)

- REST ingestion, risk score, reasons[] (+ references из политики), версия политики, маскирование номеров.
- Идемпотентность по `idempotency_key`, rate limit на ingestion.
- Деплой: Docker / Helm / метрики.
