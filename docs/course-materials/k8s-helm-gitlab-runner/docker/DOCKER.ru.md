# Описание Docker

## Лучшие практики написания Dockerfile

* Используйте официальные базовые образы (например, python:3.12-slim).
* Минимизируйте количество слоёв: каждая инструкция в Dockerfile создаёт новый слой, и слишком большое их количество может негативно сказаться на размере образа и времени сборки. (Максимальное количество слоёв в Docker по умолчанию — 127)
* Упорядочивайте инструкции эффективно: структурируйте Dockerfile таким образом, чтобы инструкции, которые изменяются реже всего, располагались в начале, а те, что изменяются чаще, — в конце. Это позволяет использовать механизм кэширования Docker для ускорения процесса сборки.
* Используйте файл .dockerignore: это помогает уменьшить размер образа и время сборки.
* Указывайте версию для каждой зависимости: для обеспечения воспроизводимости и предотвращения непредвиденных проблем.
* Используйте COPY вместо ADD, когда это возможно: COPY более явная инструкция и не пытается автоматически распаковывать сжатые файлы, как это делает ADD.
* Удаляйте ненужные пакеты: после установки пакета или выполнения определённой задачи рекомендуется удалять все ненужные пакеты или файлы, чтобы минимизировать размер образа.
* Используйте многоэтапную сборку (multi-stage builds): позволяет создавать промежуточные временные контейнеры для сборки артефактов, а затем копировать в финальный образ только необходимые файлы.
* Запускайте от имени непривилегированного пользователя: запуск контейнеров от имени пользователя, отличного от root, повышает безопасность, ограничивая доступ контейнера к хостовой системе.
* Документируйте свой Dockerfile: добавляйте комментарии и включайте понятную документацию, описывающую назначение каждой инструкции.

## Линтер для Dockerfile

Я использую Haskell Dockerfile Linter. Согласно его [документации](https://github.com/hadolint/hadolint#:~:text=A%20smarter%20Dockerfile%20linter%20that%20helps%20you%20build%20best%20practice%20Docker%20images.%20The%20linter%20parses%20the%20Dockerfile%20into%20an%20AST%20and%20performs%20rules%20on%20top%20of%20the%20AST.): «Интеллектуальный линтер для Dockerfile, который помогает создавать Docker-образы в соответствии с лучшими практиками. Линтер разбирает Dockerfile в абстрактное синтаксическое дерево (AST) и применяет правила на основе этого AST.»

Использование:

```shell
docker run --rm -i hadolint/hadolint < Dockerfile
```
